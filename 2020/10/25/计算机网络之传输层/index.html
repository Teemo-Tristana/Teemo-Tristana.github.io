计算机网络之传输层

发表于 2020-10-25

传输层：

  • 概述

    • 传输层的作用
      • 提传输层提供进程和进程间的逻辑通信， 网络层提供主机之间的逻辑通信
      • 分用和复用
    • 传输层主要有个两个协议： TCP和UDP
    • 寻址与端口：
      • 端口号 16 bit , 端口号是本地唯一标识用户进程
        • 共 2^16 - 1 = 65535 个 端口，是逻辑上的端口号
        • 知名端口: 0-1023
          • FTP 21
          • TELNET 23
          • SMTP 25
          • DNS 53
          • HTTP 80
          • HTTPS 443
        • 临时端口: 1024~65535
    • 套接字:
      • socket = 主机IP地址 + 端口号
      • 套接字socket是唯一标识网络中的主机进程

  • UDP协议(User=Datagram-Protocol)

    • UDP特点

      • UDP是基于数据报的不可靠的无连接服务

        • 数据报:
          • 每个UDP数据报都有一定长度，接收端必须以该长度为最小段位将所有内容一次性读出，否则数据将被截断
          • 发送和接受的数据报的次数是相同的
        • 不可靠
          • 尽量最努力交付“：不保证可靠交付
          • UDP无法保证数据从发送端正确地传输到目的端
        • 无连接:
          • 通信双方不建立连接，因此，每次发送数据时都需要明确指定接收端的地址字

      • 因此，使用UDP协议的应用程序需要自己处理数据确认，超时重传等机制。这些工作应该有UDP上层来完成

      • 面向报文：适合一次传输少量的网络应用

      • 无拥塞控制：适合很多实时应用：如IP电话，视频会议
